# Система стартовых вариантов и оценки риска генерации

## Стартовое меню

До первого выбора в чате нельзя запрашивать проценты, пропорции, числовую степень откровенности или другие точные параметры. Выбор показывается только нативным контекстным меню в области ввода. Оно содержит три предложенных пункта и автоматически добавляемый свободный пункт. Текстовые нумерованные меню запрещены. Если нативный элемент недоступен, автоматически и без вопроса применяется рекомендуемый профиль `90% + BODY_REFERENCE_LIBRARY`:

1. **Рекомендуемый:** `90%` соблюдения стиля, подключить `BODY_REFERENCE_LIBRARY`.
2. Контекстный вариант, сформированный ИИ для текущей задачи.
3. Другой осмысленно отличающийся контекстный вариант.
4. **Указать свой вариант.** Пользователь сразу одним сообщением полностью описывает, что требуется, включая нужные параметры.

В описании каждого типового варианта кратко показываются его художественный результат, ключевые автоматические решения и прогноз D-риска. После выбора пунктов 1-3 ИИ сам разворачивает вариант в точные параметры. После пункта 4 полное пользовательское описание принимается без дополнительных вопросов. Уточнение допустимо только при отсутствии или противоречии обязательного значения, без которого нельзя корректно подготовить план; необязательные предпочтения не переспрашиваются.

Выбор стиля и стартового профиля действует до конца текущего чата. В следующих запросах этого чата меню не показывается и вопрос не повторяется: используются ранее выбранные стиль, процент и решение по `BODY_REFERENCE_LIBRARY`, а поза, кадрирование и сцена выводятся из нового запроса. Повторный выбор выполняется только по прямой просьбе пользователя.

Постоянное изменение тела уже утверждённого персонажа не может быть скрытым параметром типового варианта. Без прямого запроса сохраняются утверждённые пропорции, а библиотека используется только для позы, камеры, поведения одежды, контактов или временного стадийного кадра.

## Шкала D1-D10

`D1` означает минимальный прогнозируемый риск отклонения, `D10` — крайне высокий. Это прогноз, а не гарантия принятия или отклонения.

Маркер ставится в конце названия или описания: `Нейтральный вид спереди [D2]`. Для референса рейтинг учитывает одновременно видимое содержание и вероятное влияние этого файла при прикреплении к следующей генерации.

## Расчёт

1. Определить базовый уровень по наиболее рискованной найденной конструкции.
2. Учесть дополнительные совместно присутствующие риск-факторы.
3. Применить только контекстно допустимые модификаторы от `-2D` до `+2D`.
4. Применить жёсткие сочетания. Например, возрастная неоднозначность вместе с сексуализацией всегда даёт `D10` независимо от смягчающих слов.
5. Рассчитать эффективный риск каждого референса: его D-рейтинг плюс влияние способа использования.
6. Взять максимум риска промта и референсов. Если приложено не менее двух референсов с эффективным `D6+`, добавить `+1D` за совместное усиление.
7. Ограничить итог диапазоном `D1-D10`.

Слова безопасного контекста применяются только когда они подтверждаются всем запросом. Например, слово «взрослый» снимает возрастную неоднозначность, но не уменьшает риск явно недопустимого действия. «Непрозрачный» не снижает риск, если другая часть промта прямо требует видимую интимную анатомию.

## Словарь

Машиночитаемый словарь находится в `config/generation_risk_lexicon.json`. Каждая запись содержит:

- идентификатор и русское название;
- контекстные русские и английские шаблоны;
- базовый D-уровень;
- вклад в сочетании с другими факторами;
- описание рискованного контекста;
- допустимые более безопасные формулировки.

Модификаторы отдельно описывают условие применения и изменение рейтинга. Новая запись не должна содержать эвфемизмы или инструкции по обходу модерации.

## Отчёт перед генерацией

`tools/generation_risk_assessor.py` создаёт JSON с исходным уровнем, найденными элементами, модификаторами, безопасными предложениями, рейтингами всех референсов и пересчётом для исправленного промта.

Пример команды PowerShell:

```powershell
python tools\generation_risk_assessor.py `
  --prompt-file .\tmp\request\PROMPT.txt `
  --revised-prompt-file .\tmp\request\PROMPT_SAFE.txt `
  --reference "D:\path\face.png::D2::0D::Нейтральное лицо; риск использования не повышается" `
  --reference "D:\path\pose.png::D5::+1D::Откровенный ракурс может усилить композиционный акцент" `
  --output .\tmp\request\RISK_ASSESSMENT.json
```

Команда анализирует исходный и исправленный промты, учитывает два референса и сохраняет проверяемый отчёт. `prepare-generation` принимает его через `--risk-assessment` и блокирует план, если хотя бы один выбранный референс не имеет D-маркера.

После фактического отклонения нельзя перебирать синонимы для того же проблемного содержания. Следует удалить или безопасно изменить сам риск-фактор, повторно рассчитать рейтинг и продолжать только с допустимой художественной постановкой.
